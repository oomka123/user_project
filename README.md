# Fullstack Authentication System

Веб-приложение с системой аутентификации, разделением ролей и личным кабинетом.

Проект состоит из двух частей:

- **Backend**: API на Django REST Framework
- **Frontend**: Клиентское приложение на Next.js

## 🌟 Основной функционал

- **Регистрация и Авторизация**: Безопасная аутентификация с использованием JWT токенов.
- **Ролевая модель**: Разделение прав доступа на обычных пользователей (`user`) и администраторов (`admin`).
- **Личный кабинет (Dashboard)**:
  - Просмотр и управление личным профилем.
  - Функция удаления собственного аккаунта.
- **Панель администратора**:
  - Доступ к списку всех зарегистрированных пользователей.
  - Сортировка таблицы по ID, имени, email и роли.
  - Возможность удаления пользователей.

## 🛠 Технологии (Tech Stack)

### Backend

- **Python 3.x**
- **Django** & **Django REST Framework (DRF)**
- **Simple JWT** — для работы с токенами
- **PostgreSQL** — надежная реляционная база данных

### Frontend

- **React** & **Next.js 14+** (App Router)
- **Tailwind CSS** — для стилизации интерфейсов
- **Axios** — для HTTP-запросов (с перехватчиками для авто-обновления токена)

### Инфраструктура

- **Docker** & **Docker Compose** — для быстрой контейнеризации и развертывания

## 🚀 Запуск проекта (Docker Compose)

Самый простой и быстрый способ запустить проект — использовать Docker. Он автоматически поднимет базу данных PostgreSQL, Backend и Frontend.

Убедитесь, что у вас установлен [Docker Desktop](https://www.docker.com/products/docker-desktop/).

В корневой директории проекта выполните команду:

```bash
docker-compose up --build
```

_После успешной сборки и запуска:_

- **Frontend** будет доступен по адресу: http://localhost:3000
- **Backend API** будет доступен по адресу: http://localhost:8000
- **PostgreSQL** будет доступна на порту `5433` (снаружи) и `5432` (внутри сети)

## 🛠 Запуск проекта без Docker (Локально)

Если вы хотите запустить сервисы вручную для разработки:

### 1. Запуск Backend-сервера

Откройте терминал и перейдите в папку `backend`:

```bash
cd backend
```

Создайте и активируйте виртуальное окружение:

```bash
python -m venv venv
source venv/bin/activate  # Для Mac/Linux
# venv\Scripts\activate   # Для Windows
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

Убедитесь, что у вас запущена PostgreSQL (или используйте SQLite для тестов, изменив настройки в `settings.py`), затем примените миграции и запустите сервер:

```bash
python manage.py migrate
python manage.py runserver
```

### 2. Запуск Frontend-приложения

Откройте новую вкладку терминала и перейдите в папку `frontend`:

```bash
cd frontend
```

Установите зависимости:

```bash
npm install
```

Запустите сервер для разработки:

```bash
npm run dev
```

## 🧪 Тестирование

Для запуска тестов backend-части (проверка аутентификации, прав доступа и регистрации):

```bash
cd backend
source venv/bin/activate
python manage.py test users.tests.UserFlowTests
```
